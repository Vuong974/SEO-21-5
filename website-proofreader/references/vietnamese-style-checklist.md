# Vietnamese Website Proofreading Checklist

## Common Issues To Flag

- Missing or wrong Vietnamese diacritics.
- Repeated words: `cua cua`, `la la`, `cho cho`, repeated phrase fragments.
- Wrong spacing around punctuation: space before comma/period, no space after punctuation, duplicated punctuation.
- Mixed casing in headings: inconsistent capitalization of service names, acronyms, and brand terms.
- Awkward sentence structure: missing subject, overloaded sentence, unclear modifier, repeated connector words.
- Inconsistent terms: `chung nhan`, `cap chung nhan`, `giay chung nhan`, `danh gia chung nhan` used inconsistently in the same context.
- English/Vietnamese mismatch: English technical terms inserted without need, or Vietnamese translation that changes the meaning of a standard.
- Broken web text: shortcode residue, HTML entity displayed to users, leftover button text, empty CTA/contact fragments.

## Terms Usually Preserved

- Standards and acronyms: ISO, HACCP, BRC, FSC, PEFC, FDA, GMP, GRS, RCS, WRAP, SMETA, BSCI, IATF, SA8000.
- Versions: `ISO 9001:2015`, `ISO 14001:2015`, `ISO 45001:2018`, `ISO/IEC 27001:2022`.
- Brand: preserve the site's chosen style for `KNA CERT` unless the user asks to normalize it.

## Suggested Report Language

- Use `Chinh ta` for spelling or diacritic issues.
- Use `Ngu phap` for sentence structure and agreement issues.
- Use `Dau cau/khoang trang` for punctuation and spacing.
- Use `Dien dat` for awkward wording or readability.
- Use `Thuat ngu` for technical term inconsistency.
- Use `Loi hien thi` for shortcode, HTML entity, or leftover website fragments.

## Review Discipline

- Quote only the short text needed to identify the issue.
- Suggest the smallest correction that fixes the problem.
- Keep SEO intent intact unless the wording is visibly wrong.
- Separate confirmed errors from optional rewrite suggestions.
